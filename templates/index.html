About FraudShield

A complete machine learning pipeline for insurance fraud detection — from raw data to a deployable web application. Built as a full academic and practical project covering data collection, EDA, model training, and deployment.

🗂 Dataset

The dataset contains 1,000 auto insurance claims with 39 features including policyholder demographics, incident details, and claim amounts. Target variable is fraud_reported (Y/N).


Source: insuranceFraud_Dataset.csv
Records: 1,000 rows × 39 columns
Fraud rate: 24.7% (imbalanced)

⚙️ ML Pipeline

1
Data Cleaning

Removed irrelevant columns, handled '?' nulls, imputed missing values

2
Encoding

Label encoding for all categorical features

3
Scaling

StandardScaler for normalization before modeling

4
Modeling

6 classifiers trained & compared on 80/20 split

📋 Models Used

Six classification algorithms were trained and evaluated:


🌳 Decision Tree (max_depth=5)
🌲 Random Forest (100 estimators)
📍 K-Nearest Neighbors (k=7)
📈 Logistic Regression
📊 Naive Bayes (Gaussian)
⚡ Support Vector Machine (RBF)

🚀 Deployment

The best model (Decision Tree, 81% accuracy) is serialized with joblib and served via a Flask web application.


Framework: Flask (Python)
Frontend: HTML5, CSS3, Vanilla JS
Serialization: joblib
Preprocessing: scikit-learn Pipeline


Run locally with: python app.py
